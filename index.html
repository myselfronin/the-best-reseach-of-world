1. Introduction

Software integration from kitchens to cars, for the benefit of human civilization, is advancing at a rapid rate. One can find software in virtually everything used in daily life. The field has stretched itself eminently by changing the way we govern, gain education, manage health, organize businesses, and so on. This means a huge amount of software has and is being developed every day. Along with the growth, the complexity of software products has also increased remarkably. The development process has become vulnerable to missing deadlines, low quality, cost overruns, and so on. Such problems or challenges must be overcome so as to prosper the whole field of software engineering toward a better future.

Along with the intricacy of software, the optimization of software phenomena or even finding effective ones has become a challenging task. Nevertheless, it is crucial for companies or organizations to continuously improve their software process in order to deliver high-quality software within a specified timeframe, budget, and with available resources. It is obligatory for companies or organizations to continuously improve their software phenomena in order to deliver high-quality software within a specified timeframe, budget, and with available resources. Thus, a systematic and disciplined approach to improving the software phenomena is needed.

This paper highlights methods that can be used to perform an evaluation in software engineering, mostly the empirical research methods. A detailed description about three different approaches to empirical studies, namely, quantitative, qualitative, and mixed methods have been discussed. Also, it presents the findings of performing surveys and interviews for evaluation of the code review process.

2. Empirical Software Engineering Evaluation

2.1 Overview

Scientific methods should be used to influence software development. Glass[1], in his paper published in 1994, has summarized four methods for research in the field of software engineering as given below.

Scientific Method: The world is observed based on which a model is built.
Engineering Method: Current solutions are studied and changes are proposed and then evaluated.
Analytical Method: Formal theory is proposed and compared with empirical observation.
Empirical Method: A model is propsed and evaluted empirically.

Although other methods are also necessary for software engineering, the focus of the paper is on using empirical strategies. Software engineering is driven by the professionals involved in software development. In order to evaluate the software phenomena, there needs a study on how software practitioners solve software problems ie discover their instincts or experience on the topic under discussion. The software phenomena can be processes, tools, techniques, policies, practices, and so on, used for developing software. Evaluating such phenomena using empirical strategies can be termed Empirical Evaluation in Software Engineering.

2.2 Empirical Strategies

Exploratory and Explanatory are two paradigms in empirical studies [2]. Exploratory is concerned with understanding the point of view and discovering the notices of the subject. The subject would mean the person participating in the empirical studies for evaluating an object. These are informed by the qualitative data. On the other hand, Explanatory is concerned with comparing two or more groups to identify cause-effect relationships. As this is informed by quantitative data, one can perform comparison and statistical analysis on them. Both quantitative and qualitative research investigations can be implemented on the same topic.

The survey, case study, and experiment are the major types of strategies, depending upon the purpose of the evaluation, whether it is techniques, methods, or tools, and depending on the conditions for the empirical investigation. Depending on the purpose of evaluation, may that be for techniques, methods, or tools, and also depending on conditions for empirical investigations, there are three major strategies [114].

2.2.1 Survey

A survey is one of the best-known methods of collecting data from a large number of target people. It is generally used for investing already in used tools or techniques and aims to generalize the findings. This empirical method can be used to collect both qualitative and quantitative data and derive explanatory and exploratory conclusions. Interviews and questionnaires are the primary means of collecting data.[wholin book]

2.2.2 Case Study

A case study is an empirical inquiry that uses a variety of sources of data to examine one or a few instances of software phenomena in its real context. It can be used to assess which of the two methods is more suited in a certain context. Research published in 2009, shows that case studies are the most frequently used in evaluating techniques in computer science [Taxanomy one].

2.2.1 Experiment

An experiment is an empirical inquiry in which one can manipulate certain factors or variables in order to study the change in the final outcome. Experiment can be human-oriented in which humans apply different treatments to objects, or technology-oriented where different tools are applied to different objects.

2.2. Datenschutz und -sicherheit

Bei solchen hohen Nutzerzahlen haben beide Anbieter eine große Verantwortung bezüglich der Sicherheit der Daten. Dazu zählen natürlich angemessene Datenschutzrichtlinien und Verschlüsselungstechniken, aber auch die Ermutigung und Aufklärung des Anwenders den eigenen Account so einbruchssicher wie möglich zu gestalten. Dies beginnt schon bei der Registrierung, nämlich mit der Festlegung eines starken Passworts.

Beide Dienste weisen bei der Registrierung auf die Stärke des Passworts mit Hilfe eines Farbbalkens hin. Die Einstufung erhöht sich je länger die Eingabe wird und je mehr zwischen Groß- und Kleinschreibung, Zahlen und Sonderzeichen variiert wird [3]. Zusätzlich wird von beiden die zweistufige Verifizierung stark empfohlen, bei der nicht nur Username und Passwort, sondern ein zusätzlicher Pin ans Handy gesendet wird, um sicherzustellen, dass es sich wirklich um den bestimmten Nutzer handelt. Aber auch hier unterscheiden sich Google und Dropbox in den Details der Umsetzung. Während Dropbox diesen Code entweder per SMS verschickt oder über eine App generieren lässt [7], bietet Google zusätzlich die Übermittlung per Sprachanruf und die Variante der Backup-Codes an [8]. Letztere sind vor allem nützlich, wenn das Smartphone bzw. der Empfang gerade nicht zur Hand ist.

Der nächste wichtige Sicherheitsaspekt ist die Übertragung der Daten. Diese findet bei beiden selbstverständlich über SSL/TSL und HTTPS statt. Danach findet die serverseitige Verschlüsselung über AES-256 bei Dropbox [7] und AES-128 bei Google [8] statt. Beide verschlüsseln die Daten jedoch mit eigenen generierten Schlüsseln, was zu der Kritik führt, dass „[s]ince Dropbox itself encrypts the data on the server-side, users cannot be sure by cryptographic means that all stored data is highly confidential“ [3]. Über Google ist die Kritik sogar noch lauter, denn hier wird erst seit gut einem Jahr serverseitig verschlüsselt [1]. Zuvor wiesen Mitarbeiter des GoogleDrive Team nur auf die sichere Kommunikation zwischen Server und Client hin bzw. machten auf den Gebrauch der zwei-Schritt-Verifizierung aufmerksam [1]. Dass die Schlüssel jedoch selbst generiert und verwaltet werden, sorgt auch hier für Unwohlsein [1].

Aus diesem Problem hat sich ein ganz eigener Markt entwickelt. Dropbox selbst weist in ihrem Helpcenter auf „zahlreiche Drittanbieter“ [7] hin, mit Hilfe welcher Anwender ihre Daten selber verschlüsseln können, bevor sie diese in die Cloud hochladen. Auch Borgmann et al. kommen zu dem Schluss, dass es „a whole range of alternative methods“[3] gibt, um die eigenen Daten schon lokal zu verschlüsseln und selber die Schlüssel zu verwalten. Einer dieser Anbieter ist Boxcryptor, 2011 gegründet und mit mittlerweile über 1 Millionen Downloads oft genutzt [4].

Boxcryptor ist eine Anwendung die speziell zur Verschlüsselung von Daten, welche anschließend in der Cloud gespeichert werden sollen, entwickelt wurde. Das Prinzip ist recht simpel. Die zu verschlüsselnden Daten werden in einem entsprechenden Ordner auf einem virtuellen Laufwerk abgelegt und dort mit Hilfe der AES-256 und RSA Algorithmen verschlüsselt. Danach erst werden die Dateien in den Cloud Ordner weitergegeben. Verschlüsselte Daten sind farblich markiert und leicht von anderen, nicht verschlüsselten Daten, zu unterscheiden. Diese Software ist kompatibel mit allen großen Cloud Dienst Anbietern und allen Betriebssystemen. Auch für Android, iOS und Blackberry gibt es Apps, sodass hier ebenfalls keine Einschränkung besteht [4].

3. Alternativen

3.1. BitTorrent Sync

Neben zusätzlichen Verschlüsselungssoftwares, die mit Cloud Diensten zusammen angewendet werden können, haben sich aber auch Dienste entwickelt, die auf Serverspeicherung gänzlich verzichten, und dennoch die gleichen Vorteile versprechen. Zwar noch in der Beta Phase, aber trotzdem ein vielversprechendes Konzept, präsentiert beispielsweise BitTorrent Sync. Hier werden Daten direkt zwischen Geräten synchronisiert. Dazu müssen diese in dem entsprechenden Sync Ordner abgelegt werden, woraufhin ein 20 Byte langer Geheimschlüssel generiert wird. Über diesen Schlüssel können anschließend andere Geräte auf diese Daten zugreifen. Dies ermöglicht ebenfalls das Teilen von Daten mit Dritten. Mit Hilfe drei verschiedener Einstufungen der Geheimschlüssel, auch Secrets genannt, ist es möglich, Berechtigungen zuzuschreiben und sogar ein zeitliches Limit des Teilens zu setzen [2].

Ein entscheidender Vorteil ist, dass durch dieses Prinzip keine Einschränkung des zu verfügbaren Speicherplatzes seitens des Anbieters stattfindet. Dieser richtet sich nach der eigenen Kapazität. Außerdem entstehen keine Fragen bezüglich des Standortes der Server und damit folgenden unterschiedlichen Datenschutzrichtlinien, da die Daten nur auf den eigenen Geräten gespeichert werden. Gerade in Hinblick auf die Abhörskandale der NSA und auch des BND stößt dies auf große Begeisterung:

„That difference in design means that people using Bittorrent Sync don’t have to worry about whether the cloud company hosting their data is properly securing it against rogue employees or other threats. Forgoing the cloud also means that data shared using Bittorrent Sync could be harvested by the NSA or another agency only by going directly to the person or company controlling the synced devices”[11].

Die Kommunikation zwischen zwei Geräten ist über den AES-128 Algorithmus verschlüsselt. Zudem wird das P2P Protocol verwendet. Trotz Beta Phase erstreckt sich das Angebot der Desktop Version bereits über Windows, Mac und Linux, und auch die Unterstützung mobiler Geräte deckt Android, iOS, Windows Phone und den Kindle Fire ab [2].

Ein für diese Technik spezifischer Nachteil ist allerdings die Voraussetzung, dass zur Synchronisation die entsprechenden Geräte angeschaltet sein müssen[11]. Sollte die Verbindung zu einem Zeitpunkt der Übertragung abbrechen, wird diese jedoch an der entsprechenden Stelle wieder fortgesetzt, sobald beide Geräte wieder online sind[6].

3.2 ownCloud

Wer dennoch lieber eine Cloud zur Hand haben und trotzdem auf einen kommerziellen Anbieter verzichten möchte, hat die Möglichkeit mit ownCloud sich selber eine Cloud aufzusetzen. Zuerst vorgestellt in 2011, erfreut sich diese Software heute recht großer Beliebtheit. Nach eigenen Angaben nutzen mehr als 1,3 Millionen Anwender diesen Service [10]. Und tatsächlich finden sich zahlreiche Anleitungen im Internet, wie man sich zu Hause mit Hilfe eines Raspberry Pis und ownCloud selber eine Cloud basteln kann [12] [9] [5].

Die Vorteile sind schnell zusammengefasst: Erstens orientiert sich die Speicherkapazität, ebenso wie bei BitTorrent Sync, an den eigenen Gegebenheiten. Der Vorteil gegenüber dem Konzept von BitTorrent Sync ist allerdings, dass durch die Zwischenspeicherung der Daten, kein anderes Gerät als das auf dem die Daten gewollt sind online sein muss. Vor allem aber ist der Ort der gespeicherten Daten selbst bestimmt.

Jedoch ganz ohne Schattenseite ist auch diese Alternative nicht. Gewisse Kenntnisse über Server, Netzwerke und Sicherheitsprotokolle sollten vorhanden sein [5]. Zudem ist die Hochverfügbarkeit, wie sie bei Dropbox oder Google herrscht, nicht gewährt.

5. Fazit

Die Vorteile der Nutzung von Cloud Diensten liegt klar auf der Hand: Die Daten sind unabhängig von Standort und Gerät immer abrufbar. Im Falle eines Schadens an der Festplatte oder ähnlichem sind zudem diese Daten nicht verloren. Auch vereinfacht es Projektarbeiten. Doch es gibt immer zwei Seiten einer Medaille und so sollte sich jeder bewusst sein, dass wenn man seine Daten, seien es persönliche, geschäftliche, belanglose oder sensible Daten, online hochlädt und auf fremden Servern speichert, die theoretische Möglichkeit besteht, dass jemand Fremdes sich leichter und unter Umständen unbemerkt Zugriff zu diesen Daten beschaffen kann.

Große Anbieter wie Dropbox und Google versprechen viel Leistung, gerade auch in Hinblick auf Sicherheit, aber so zeigte sich besonders im letzten Jahr wie viel hinter den Kulissen geschieht, ausgehend von der Politik und den Geheimdiensten, von dem Anwender nichts mitbekommen. Die besten Verschlüsselungsalgorithmen bei der Übertragung von Daten oder beim Ablegen der selbigen auf den Servern nützen nichts, wenn die Schlüssel von den Firmen selbst verwaltet werden und sie auf Verlangen der Regierung diese herausgeben müssen.

Eine Alternative ist sicher die zusätzliche lokale Verschlüsselung, so wie sie Boxcryptor anbietet. Oder auch gänzlich auf das Zwischenspeichern auf Servern zu verzichten und Daten direkt zu Synchronisieren, so wie BitTorrent Sync es vorschlägt. Die Idee sich seine eigene Cloud aufzusetzen erscheint im ersten Moment auch eine interessante Alternative zu sein. Jedoch kann dieser Enthusiasmus schnell verfliegen, denn die Herausforderung diese so sicher wie möglich zu gestalten und den gleichen Service zu bieten wie die Vorbilder, ist doch größer als anfänglich gedacht. Viel Freiheit in Gestaltung und Umsetzung bedeutet in diesem Fall auch sehr viel Verantwortung, welche nur mit Grundkenntnissen kaum erfüllbar ist.

1. Three Research Approaches

In this paper, we are going to go deeper into three research approaches: quantitative, qualitative and mixed methods. Mostly, quantitative and qualitative approaches are seen as opposite sides or distinct categories. Instead, they represent different ends of a continuity so a study can be more quantitative in some aspects than qualitative or the other way. For some cases, a mixed method research is needed to incorporate elements of both quantitative and qualitative. [todo page 41 first paragraph]

Traditionally, quantitative and qualitative research are differentiated by the idea that quantitative research uses numbers instead of words used by qualitative research or that quantitative research uses closed-ended questions and qualitative research uses open-ended questions. This is indeed true but a more complete way to see the differences between these two approaches is to look at the types of research strategies used(e.g, surveys, experiments, interviews), philosophical assumptions brought by the researcher and the method or tools used to conduct these approaches (e.g., instruments used to collect quantitative data or observations used to collect qualitative data). Quantitative research has dominated the research approaches from the late 19th to mid-20th century. After that period of time, the interest for qualitative research approaches increased, thus growing the interest for mixed research methods. [todo page 41 second paragraph]

2 Quantitative Method

Quantitative research approach uses objective theories and tests them by analyzing different variables and the relationship between them. These kinds of variables are measured using instruments so numerical data can be collected and used to produce statistical results. The researcher in this approach builds the research against the bias, eliminates or controls new nuances of explanations and findings, and builds a research that can be replicated and generalized. [todo page 41]

In a quantitative approach, the researcher avoids the variation during the introduction of the topic since the objective is to what factors and variables influence an outcome and not to generate new ideas or solutions based on the participants perspective. For example, quantitative research can be conducted to understand if the financial issues contribute to a divorce. In this case, the variables and factors help the research to understand and explain this particular problem. The quantitative research introduction should not be written and affected by the personal view of the researcher in order to support objectivity. [todo page 162]

2.1 Quantitative Methodologies

The researcher, despite the fact that has to choose between the quantitative, qualitative and mixed research approach, also decides on what type of design to choose within these approaches. Research methodologies are different kinds of inquiry that provide instructions and direction for procedures to be followed in a study. These types of design have grown dramatically over the years since the advances made in technology and tools to collect and analyze complex data. [todo page 49]

2.1.1 Field experiments

Field experiments are conducted in actual environments. To test the effect, one or more factors must be isolated and changed. Although there are many more variables to take into account, it enables the researcher to witness more natural behavior. This method is also frequently used in applied sciences like bioengineering and medicine as well as sociology. When contrasted to lab tests, field experiments provide a number of advantages. It provides a setting that is more similar to nature than a lab. It is appropriate to examine large groups of people since they tend to be more representative. On the other side, controlling variables is more complicated, making it very difficult to reproduce the same conditions. Furthermore, because it is challenging to manage the environment, unforeseen events may occur and have a negative impact on patients and participants. [todo papger page 380]

2.1.2 Surveys

A survey is a research method that enables the direct gathering of data from a participant in the study using a series of questions arranged in a certain order. It is one of the most popular quantitative techniques because it enables the gathering of data on a particular topic by asking questions that capture the attitudes, perceptions, and actions of a group of people. Surveys have a number of advantages. The high population representativeness and the method's inexpensive cost in comparison to other options are two of its most significant advantages. On the other hand, the survey's design and the respondents' accuracy of response depend greatly on the validity of the data. [todo paper page 381]

2.1.3 Correlation study

In essence, correlational research is an experimental technique used to examine if there is a correlation between two or more variables. There is merely an examination of how closely the variables are related; there is no manipulation of the variables. A correlational analysis emphasizes two characteristics: the relationship's strength and direction. We can quickly quantify observational data by looking at the correlation coefficient. At the same time, we can gather a variety of data from numerous domains at once and investigate the relationships between those variables. The link between two variables could perhaps be explained by a third variable, hence correlation does not imply causality. [todo paper page 381]

3. Qualitative Method

Qualitative research approach is used for understanding the perspective of individuals and their experience on a social problem and exploring a topic for different nuances. The process of conducting this kind of research approach involves data collected from participants in different life situations and experience which is represented by the interpretation of the researcher. The focus is on the individual meaning rather than numbers and statistics. [todo page 41]

In a qualitative approach, the problem that the researcher describes can be understood by exploring a topic and comprehending new variations given by the participants. Qualitative research problem has some characteristics: the topic has a lack of previous research and theory which may be incorrect, inaccurate or biased; the topic needs to be developed further; a quantitative approach is not suitable for this kind of problem. In a qualitative approach, the researcher should rely on the participant's perspective and may begin the introduction of the problem with a personal opinion or experience from the researcher in order to focus on the subjective point of view. [todo page 162]

3.1 Qualitative Methodologies

In qualitative approaches, the design types became more visible and clear during the 1990s. The origins for these approaches come from sociology, humanities, and evaluation. There are many methodologies and design types for qualitative research but we will focus on observations, ethnography and structured interviews and show their strengths and weaknesses. [ todo page 50]

3.1.1 Observation

Researchers examine a specific phenomena in its natural setting as part of a systematic method for gathering information called observation. This approach is particularly recommended when a subject is largely unknown and it is crucial to fully comprehend a phenomenon while preserving the surrounding circumstances. Observation is a useful method for gathering information concurrently with the occurrence of the event without interfering with it. It is a discrete, highly adaptable strategy that is focused on knowledge discovery. In some circumstances, such as the study of human and animal behavior, trustworthy data collection is the only option. On the other hand, it is a very time-consuming strategy that needs advance planning and the researcher's availability to travel to the location where the event occurs. Additionally, since the researcher is the only one who interprets the data, the method is extremely sensitive to the independence of the analysis. [todo paper page 376]

3.1.2 Ethnography

In ethnography, people are interviewed after being observed in a circumstance. In an ethnographic study, the researcher makes an effort to understand the scenario being observed from the participants' points of view. There are two fundamental elements of ethnography: (i) the observation occurs in a natural context, and (ii) researchers must comprehend how an event is experienced and processed by members of a speech community. The methodologies of observation and ethnography are extremely similar. However, when doing an ethnographic study, the researcher must adopt a more comprehensive perspective and pay close attention to every detail.[todo paper page 376]

The main benefit of ethnography is that it allows the researcher to gain a thorough understanding of the issue under study. However, ethnography involves a significant time commitment from the researcher, and because the study's findings can be so varied, it can be challenging to draw accurate and focused conclusions.

3.1.3 Structured Interviews

Structured interviews are a type of assessment tool used to collect and compare interviewee replies. The interviewee is questioned about prior encounters and/or suggested fictitious settings. All interviewees go through the same procedure. High response rates are normal for structured interviews, and the interviewer is present to explain the question to prevent respondents from misinterpreting it. A systematic interview might take time to prepare, though. Additionally, even if during the interview new lines of study sound fascinating to pursue, they do not have the same flexibility in the choice of the responses that are predefined and cannot be changed by the researcher.[todo paper page 377]

4. Mixed Method

Mixed method research approach is used to investigate topics that need to collect both qualitative and quantitative data and to integrate these data to come to fully described conclusions supported by both types of data. The main objective of this approach is to give additional insight for a topic that cannot be given by either quantitative or qualitative data alone. [todo page 41-42]

In order to write the introduction for a mixed method study, the researcher can either put an emphasis on the quantitative or qualitative approach and maybe some combination of both. This emphasis will be shown in the introduction of the study. For studies that need to have an equal emphasis on quantitative and qualitative, there is a necessity to understand the topic in both quantitative and qualitative data in order to show the relationship between variables and explore the topic in detail. A mixed method approach can be primarily quantitative or qualitative when there is a need to make the study more diverse in terms of methodologies and to be more inclusive. For example, a primarily quantitative research may need to explain the relationship between smoking behavior and depression and then dive deep into the detailed views and perspectives of these participants. [todo page 163]

4.1 Mixed Method Methodologies

Mixed methods combine quantitative and qualitative research and the data collected from both of them. Quantitative data contain close-ended responses from participants found on questionnaires and surveys while qualitative data contains open-ended responses which can result in different nuances of the topic. Both of these methods have their weaknesses and bias so the collection of them will result in neutralizing these weaknesses by supporting each other. Some of the best ways to use these two types of data are: (a) the validity of a database can be checked or verified by the other database; (b) one database can explain the findings of the other database; (c) one database can use the other database and build on that.[todo page 51]

Convergent mixed methods is a design type where the researcher collects both forms of data at the same time and merges the information to analyze it and interpret it as a whole. If there is any contradiction or conflict while integrating the data, they are explained by the researcher.[todo page 52]

Explanatory sequential mixed methods is a design type in which the researcher conducts the quantitative session and then the qualitative one. The quantitative data are analyzed and the findings are explained by the qualitative responses. This design is explanatory because the quantitative data are explained by qualitative data and it is called sequential because the two types of research are conducted one after the other. This design is used mostly in topics with a strong quantitative direction where the quantitative results need to be further explored by qualitative data.[todo page 52]

Exploratory sequential mixed methods is the opposite of the explanatory. The qualitative research is conducted first in order to get the participants' views and analyze them. With this information, the researcher can choose the perfect instrument to conduct the quantitative part. The analyzed data allows the researcher to check for appropriate instruments and find the variables that need to go into the quantitative session.[todo page 52]

5 Demo

In this research, we are conducting a quantitative and qualitative method about the code review process. Code review is an important process for software evaluation and improvement nowadays. This process is relatively new and some more insight is needed on what developers think about this process by conducting interviews for the qualitative part. In order to support this data, we conducted surveys and analyzed the results to provide some statistical information.

The hypothesis of this evaluation is that the code review process is extremely important so we need to prove this by quantitative data and also get some detailed responses and experiences from professionals related to the software industry. We also want to find what tools and methods are more common in the code review process. The qualitative session will also provide us with different nuances of what code review process means from the participants’ perspective.

5.1 Audience

We are focusing our research on professionals in the software industry with different levels of experience, from junior to senior. This allows us to see how the code review process is perceived by someone who is just starting in the software world and how it can evolve as a concept during the experience growth. Another important point to investigate is to relate the importance of the code review process with experience, this shows how this process is presented to the juniors and if the importance is increased or decreased throughout the years.

5.2 Questions of Survey

Q1: Have you ever involved yourself in a code inspection to find common vulnerabilities, possible performance issues, possible code improvements and improve coding style?

Q2: Has this process helped you improve the code quality?

Q3: What type of code review process have you used?

Q4: If you have a working code, will you use code review for performance improvement and implement the changes afterwards even if the deadline has almost passed?

Q5: Have you ever reviewed the code of your colleagues or vice versa? If yes, how effective did you find peer code review?

Q6: Have you ever used automated code review tools? (For example: plugins in IDE like ESLint, softwares like SonarQube)? If yes, how effective did you find automated code review?

Q7: After the code review, some coders find themselves investing a few minutes to some hours of additional time, whereas others find investing more time than it had taken them to program before the review. In scale of percentage how much time have you found yourself investing (generally) for modification of your work? Note 100% is the total time taken for you to come up with a workable solution. (Note 100% = total time taken to complete the feature)

Q8: If implementing the code review changes takes more than the time needed to implement the new feature, we can say that the code review process is not effective. How much time do you think code review changes should take in order to be effective? (Note 100% = total time taken to complete the feature)

We start the survey with the question that asks the participants if they have ever been part of the code review. This questions’ objective is to give more insight into the code review process so the participants’ will have a clear idea of what this process is (Q1). The questions Q2 - Q4 are used to get an idea of how important do the participants think that code review process is and check if they find it as a critical process or not (Q4).

We ask the participants if they have ever used peer review or automated review and see how effective they find it (Q5 and Q6). These questions are used to compare the two most used code review types and fulfill one of our objectives to see which tools and methods are most common in the code review process.

In Q7, we ask participants how much time it takes to implement the changes after a code review session. Our intention with this question is to indirectly see how important and serious is the code review process because if it takes a huge amount of time then it means that this process is used for serious purposes not only for best coding practices and styles.

We included a biased question (Q8) to see if the participants have a clear understanding of the importance of code review or not. We say that if the code review changes take up a lot of time, then the code review process is not effective and then we ask them to give a reasonable time range. If they give a high range then we can say that the code review process is seen as extremely important by the developers.
